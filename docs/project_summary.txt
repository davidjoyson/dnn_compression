DNN Compression — Project Summary
===================================
(Last updated 2026-07-22 — see docs/experiment_log.md for full history)

WHAT IT IS
----------
A research project testing whether a biologically-inspired dendritic neural network can be
compressed 4x with no accuracy loss on edge-AI classification tasks (wearables, sensors,
biomedical signals), and whether its branching structure gives it an actual robustness
advantage under quantization over conventional MLPs.

CORE HYPOTHESIS — STATUS: PARTIALLY CONFIRMED, CENTRAL CLAIM NOT SUPPORTED
---------------------------------------------------------------------------
The dendritic network's "snowflake" property — parallel branches each learning distinct
feature subspaces — was hypothesised to make it especially compatible with per-layer int8
quantization, since each branch's weight distribution was expected to be narrow and
independently calibrated, giving it better compression tolerance than a conventional MLP.

  - CONFIRMED: branches really are narrower and more weight-distinct than a structurally
    matched non-branching control layer (17-27% lower weight std, near-zero inter-branch
    cosine similarity, all 3 datasets).
  - NOT CONFIRMED: that narrowness does NOT translate into a measurable quantization-
    robustness advantage. A full 10-seed, 3-model (Dendritic vs. total-param-matched MLP
    vs. per-layer-matched MLP), 8-method comparison found an identical ~0.0026 mean
    accuracy-delta under compression across all three architectures. The motivating
    "outperforms standard quantization baselines" claim is not supported by the data.
  - WHAT DOES HOLD UP: Dendritic's plain, uncompressed accuracy edge over both baselines
    on the harder tasks (ECG, HAPT) — a real result, just independent of quantization.

ARCHITECTURE
------------
DendriticNetwork:
  Input -> FC1 -> N parallel branches (each FC+ReLU) -> Concat -> Soma -> FC2 -> Output

  Soma: collapses each branch's activations to one signal per branch (biologically motivated
  dendritic integration). Compared against two baselines: MLPBaseline (single hidden layer,
  total parameter count matched) and LayerMatchedMLP (per-layer shape matched — added
  2026-07-18 after the total-param-only comparison was found to hide very different weight
  matrix shapes behind an equal total).

COMPRESSION METHODS TESTED (8, all vs. the same uncompressed baseline)
-------------------------------------------------------------------------
  Snowflake int8        Per-layer int8, one scale per layer group (weight+bias)   4x
  Global int8           Single scale across all parameters                        4x
  Dynamic int8          PyTorch quantize_dynamic on Linear layers                ~4x
  Static int8           Per-tensor static calibration (prepare/convert)          ~4x
  Per-channel int8      One scale per output neuron row; biases stay float32     ~4x
  QAT int8              Quantization-aware training (prepare_qat_fx/convert_fx)  ~4x
  Snowflake+Static int8 Snowflake weights + true static-quantized activations    ~4x
  Mixed precision       Inner layers int8, first/last layers float32            ~0.9x
  (Snowflake int4 also implemented; viability is model-size-dependent, not
  re-verified on the current 3-dataset default set since EEG was dropped.)

  Real hardware distinction (Raspberry Pi 3, see EDGE DEPLOYMENT below): Snowflake is
  weight-only (dequantizes to float32 before compute, ~1.0x real speedup despite 4x
  storage savings). Static/Snowflake+Static/QAT run true int8 arithmetic and deliver a
  genuine ~1.8-2.2x latency reduction. Storage compression and compute speedup are not
  the same thing.

KEY RESULTS (50 epochs, 10 seeds: 42,0,7,1,2,3,4,5,6,8)
-----------------------------------------------------------
  Dataset                       Classes  Uncompressed      Snowflake int8     Delta    TOST
  UCI HAR                       6        94.12% +/-0.48%  94.16% +/-0.45%   +0.04%   EQUIV
  ECG Heartbeat (patient-split) 5        87.92% +/-1.86%  88.38% +/-1.48%   +0.46%   EQUIV
  HAPT (transitions incl.)      12       92.51% +/-0.74%  92.77% +/-0.56%   +0.26%   EQUIV

  All 24/24 method-dataset pairs are TOST-equivalent (n=10, +/-2% margin).
  Snowflake matches or beats uncompressed on all 3 datasets.

  NOTE ON DATASETS: EEG Brainwave (4th dataset, 3-class emotion) was dropped 2026-07-21
  after investigation confirmed unfixable patient/session-level data leakage in the source
  CSV. ECG uses a patient-independent (DS1/DS2) split, not the original Kaggle random
  split — the original was found to leak patient data between train/test, inflating
  accuracy by ~13pp. ECG and HAPT both train on their natural, unbalanced class
  distribution (balance=False) — oversampling minority classes by duplication was tried
  and reverted (net negative: less consistent TOST result, no real F1 improvement).

KEY FINDINGS
------------
1. Snowflake int8 = ~4x compression, no statistically significant accuracy loss (24/24
   TOST-equivalent). Per-layer calibration matters; global int8 is slightly less reliable.

2. Dendritic shows NO consistent quantization-robustness advantage over either MLP
   baseline (identical ~0.0026 mean accuracy-delta across all 3 models, all 8 methods,
   10 seeds). This is the project's central negative result — see CORE HYPOTHESIS above.

3. Dendritic's branches are measurably narrower and more weight-distinct than a
   structurally-equivalent non-branching control layer (17-27% lower weight std). Real
   structural property, but it doesn't confer the anticipated compression benefit.

4. ECG's macro F1 / balanced accuracy remain low (~0.37 / ~0.39) regardless of training
   approach. Four independent techniques tried and failed to improve it: oversampling by
   duplication, class-weighted loss, focal loss, Tversky loss. Root cause is data scarcity
   (8 real "Unknown"-class training examples, not a fixable training-methodology gap).
   Investigation closed 2026-07-21.

5. Dynamic int8 is the least reliable method, and its real-hardware speedup verdict
   flips with batch size: slower than float32 at full-throughput batching (0.44-0.60x)
   but faster at batch=1 (1.27-1.40x) — deployment batch size determines whether it helps.

6. Topology sharing (forcing branches to share weights) is destructive — collapses
   accuracy. Branch diversity is load-bearing for whatever the architecture does well.

7. On real Raspberry Pi 3 hardware: true-int8 methods (Static, Snowflake+Static, QAT)
   give ~1.8-2.2x real latency speedup; Snowflake (weight-only) gives ~1.0x despite 4x
   storage compression. Memory usage does not trade off consistently against speed
   (dataset-dependent, not a universal rule). No thermal throttling risk under 15 min
   sustained load (steady-state 46.2C, no active cooling).

8. Architecture-size ablation (h1=16/32/64) shows the expected tiny-model capacity cliff
   on HAR and HAPT (accuracy craters with huge seed variance) but it's nearly invisible
   on ECG in raw accuracy — because ECG's imbalanced test set lets a degenerate model
   score ~89% by always predicting the majority class, masking the same underlying
   capacity failure (same accuracy-vs-F1 blind spot as finding 4).

DATASETS (3 active)
--------------------
  UCI HAR            561 features, 6-class wearable activity (walking, sitting, etc.)
  ECG Heartbeat       187 features, 5-class arrhythmia, patient-independent DS1/DS2 split
                      (MIT-BIH via PhysioNet), natural unbalanced distribution
  HAPT                561 features, 12-class activities + postural transitions,
                      natural unbalanced distribution
  EEG Brainwave       DROPPED 2026-07-21 — unfixable patient/session leakage in the only
                      available source data. Loader/experiment code kept for reference,
                      not wired into the CLI.

WHAT WAS EXPLORED AND RULED OUT
--------------------------------
  Topology sharing              Always destructive — validated by ablation
  Original/leaky ECG split      ~13pp accuracy inflation from patient leakage — replaced
                                 by patient-independent DS1/DS2 split
  EEG dataset                   Unfixable leakage in the only available source data —
                                 dropped from the default experiment set entirely
  Oversampling by duplication   Net negative for ECG/HAPT — less consistent TOST result,
                                 no real F1/balanced-accuracy improvement — reverted
  Class-weighted loss           No F1 improvement (recall up, precision collapsed
                                 equally) — reverted, not committed
  Focal loss, Tversky loss      Both tried for ECG rare-class F1 — neither beat plain
                                 CrossEntropy; Tversky backfired (0 recall on 3 classes)
  Pruning 75% (historical)      Catastrophic at ~17k params — dropped early in the project
  TensorFlow port (historical)  Built, verified parity with PyTorch, then deleted
                                 (project stays PyTorch-only)
